## 1. 概要

デイジー東京の講習会で習う方法では、PLEXTALK Producer というソフトウェアを使って、テキストの取り込み、音声の取り込みまたは録音、デイジー編集、出力という一連の作業を行っている。しかしこのソフトウェアは高額であり、習得も難しいため、この手順のままでは校正等の作業に関われる人員が不足する。

マルチメディアデイジー図書を図書館に納品するためには、一定の品質を保つ必要がある。そのためには、多くの人員が関われるような作業手順を確立する必要がある。このページでは、現状の設備と人員で可能な作業手順を説明する。

この手順では PLEXTALK Producer を利用できない。なぜなら、 PLEXTALK Producer は epubインポートする際に、他のソフトウェアで作成した epub内のハイライト分割を認識できないからである。 PLEXTALK Producerのこの仕様はテキストチームの仕事の一部を無駄にしてしまうため、現状のやまびこのチームワークには利用できない。テキストチームの作業を PLEXTALK Producer 習得者だけで完結するならば、 PLEXTALK Producer も利用可能である。

マルチメディアデイジーの形式としては EPUB3 のみを作成し、 DAISY2.02 は作成しない。その理由は、小中学生に配布されるデイジー教科書の形式が今年度からすべて EPUB3 になったことと、 EPUB3 の方が DAISY2.02 よりも、[読み飛ばし機能 (skippability) や回避機能 (escapability) ](https://www.w3.org/publishing/epub3/epub-mediaoverlays.html#sec-behaviors-skip-escape)、文字エンコーディングの点で利用しやすいことである。

ただし、やまびこから EPUB3 マルチメディアデイジー図書を納品するときは、同時に DAISY2.02 音声デイジー図書も納品する。その理由として以下のことが挙げられる。

1. 非常に古い型の音声デイジー再生機では、 EPUB3 を再生できない。
2. EPUB3 仕様では、グループ機能の代わりに読み飛ばし機能や回避機能があるため、グループ機能は存在しない。しかし、特定の場所に戻って繰り返し聞きたい場合など、グループ機能がある方が便利なこともある。そういう利用方法に対応するため、 DAISY2.02 音声デイジー図書をグループ機能付きで作成する。

「EPUB3音声あり」と「音声DAISY2.02」では凡例で言うべきことが異なるので、「EPUB3音声あり」用の「デイジー図書凡例」と、「音声DAISY2.02」用の「録音図書凡例」の両方を、セクションレベル1として録音する。最後に EPUB3 と DAISY2.02 を完成させるときには、それぞれの不要な方の凡例を削除する。

DAISY2.02 の製作では、どんなに複雑な書籍でもセクションレベルを最大3までに抑えるのが暗黙の了解になっている。その理由は、利用者が使わないので付けないよう要求しているからと教えられている。 使わないからといってそれを禁じる必要は本来ないはずだが、そういう要求が出る原因の1つとして、再生機の仕様の問題が考えられる。普及している DAISY2.02 再生機では、セクション階層切り替えボタンを使ってグループ機能に切り替えるため、セクションレベルが多いとグループ機能の利用にたどり着くまでの操作が煩雑になる。

一方、 EPUB3 にはグループ機能が無い。セクションレベルが多くても、深い階層を使いたくない人は使わずに済ませば良いので、セクションレベルを 4以上にすることに再生操作上の問題があるとは考えにくい。ただし、階層が増えればどんな利用者にとってもわかりにくい構造になることは考慮する必要がある。基本的には、原本書籍の目次どおりの階層を付けるのが無難と思われる。EPUB形式上はレベル6までの階層を付けることが可能になっている。


### 1.1. 音訳表に記入

プロジェクトリーダー（やまびこでは基本的に読み手またはChatty編集者、分担する場合は統括する人）は、[音訳票の最新版 onyakuhyo\*.xlsx.zip をダウンロードする](https://drive.google.com/drive/folders/1lQOdyBPgECghazdSIF-zWrg5ktukqLJa?usp=sharing)。ファイル名の数字が大きいほうが新しい。ダウンロードしたファイルを展開すると xlsx ファイルが出てくる。その xlsx ファイルを、自分で区別しやすい名前に変更する。

その xlsx ファイルを、 Microsoft Excel, Google Sheets, LibreOffice Calc などの好きな表計算アプリで開き、わかるところに記入して、このデイジー製作に関わる人々で共有する。 記入した音訳票を自分の Google ドライブに保存すると更新や共有が簡単になるが、共有の際は、特定のアカウントからしかアクセスできないように注意する（依頼者の個人情報保護のため）。やり方がわからない場合は、作業ファイルと一緒にUSBメモリやメール添付で共有する。

第1面に記入すると第2面が自動的に埋まる。プロジェクト完了時に、第2面を印刷して制作部（やまびこキャビネットの下の段、左の扉の裏のポケット）に提出する。印刷には、デジタル工房にあるプリンタと、やまびこキャビネットの下の段にある紙を使っても良い。


### 1.2. 人間の声が入ったマルチメディアデイジー製作の概要

使用する機器やソフトウェアは、以下のとおり。

1. [PRSPro](http://www.plextalk.com/jp/products/prspro/) または [DR-1](http://www.plextalk.com/jp/support/dr1/)
2. スキャナ（デジタル書籍が無い場合）
3. 製品版OCR： [読取革命 Ver.15](https://www.panasonic.com/jp/company/pstc/products/yomikaku.html) または [e.Typist v.15.0](https://mediadrive.jp/products/et)（デジタル書籍が無い場合）
4. テキストエディタ（デジタル書籍が無い場合。 メモ帳、 ワードパッド、 Google Docs、 Microsoft Word、 LibreOffice Writer、 Vim、 Emacs などのうち、好きなもの）
5. [ChattyInfty3](https://www.sciaccess.net/jp/ChattyInfty/index.html) (AITalk版でもSAPI5版でも良い)
6. [Tobi](https://daisy.org/activities/software/tobi/)
7. 自作の自動修正スクリプト（Linuxで使えるものしか作っていない）
8. [pagina EPUB-Checker](https://www.pagina.gmbh/produkte/epub-checker/)
9. [Ace by DAISY](https://daisy.github.io/ace/)

音声チームとテキストチームで分担し、以下の手順で製作する。

1. 音声チームは、音訳者、校正者、音声デイジー編集者から成る。  
  PRSProやDR-1を使って、人間の声の音声デイジー図書を作る。  
  PCMデータのままで以下の3の作業者に渡す。
2. テキストチームは、Chatty編集者、Chatty校正者から成る。  
  原本のデジタル書籍が無い場合は、スキャン者、OCR者、テキスト修正者も必要。  
  ChattyInfty3を使って編集する。ただし合成音声の読み上げの修正は不要。   
  Chatty のimlxファイルと、それに付随する画像等（imlxファイルを置いたフォルダの中に自動的に作られる）を、以下の3の作業者に渡す。
3. 次に、Tobi作業者が 2つのチームの成果を合成する。  
  テキストチームからのimlxファイルをChattyで開いて、EPUB3（音声あり）としてエクスポートし、それをTobiでインポートする。そこに入っている合成音声の音声データを削除し、音声チームからのフォルダに入っている人間の声の音声データを割り当てる。  
  エクスポートはしなくて良い。  
  各チームから受け取ったフォルダと、Tobiで作成した作業データのすべてを、最後の作業者に渡す。
4. 最後の作業者は、DAISY規格やEPUB3規格、EPUB Media Overlays規格、WCAG、WAI-ARIA 等をある程度理解している必要がある。  
  受け取ったTobiのデータから、EPUB3（音声あり）、mp3, 64kbpsとしてエクスポートする。そのEPUB3ファイルはWCAGの基準を満たしていない。  
  自作の自動修正スクリプト等を使ってデータを改変することによって、アクセシブルなデジタル図書としての形式を整える。  
  pagina EPUB-Checker と Ace by DAISY を使って、図書館に納める製品として間違いのないマルチメディアデイジー EPUB3（音声あり）になっていることを確認する。  
  音声デイジーのデータも PRSPro で開き、枠アナウンス等の形式が納品用として正しいことを確認してから、mp3, 64kbpsとして音声エクスポートして完成させる。

### 1.3. AITalkの声が入ったマルチメディアデイジー製作の概要

使用する機器やソフトウェアは、以下のとおり。

1. スキャナ（デジタル書籍が無い場合）
2. 製品版OCR： [読取革命 Ver.15](https://www.panasonic.com/jp/company/pstc/products/yomikaku.html) または [e.Typist v.15.0](https://mediadrive.jp/products/et)（デジタル書籍が無い場合）
3. テキストエディタ（デジタル書籍が無い場合。 メモ帳、 ワードパッド、 Google Docs、 Microsoft Word、 LibreOffice Writer、 Vim、 Emacs などのうち、好きなもの）
4. [ChattyInfty3](https://www.sciaccess.net/jp/ChattyInfty/index.html) (AITalk版。英語以外の外国語を含む場合はSAPI5版を併用。)
5. 自作の自動修正スクリプト（Linuxで使えるものしか作っていない）
6. [pagina EPUB-Checker](https://www.pagina.gmbh/produkte/epub-checker/)
7. [Ace by DAISY](https://daisy.github.io/ace/)

合成音声でのデイジー版製作では、上の 1.1節の手順として示した 4ステップのうち、2と4に相当するステップだけで完成する。ただし、人間の声を使わない代わりに、合成音声の読み上げを調整する必要がある。

1. テキストチームは、Chatty編集者、Chatty校正者から成る。  
  原本のデジタル版が無い場合は、スキャン者、OCR者、テキスト修正者も必要。  
  ChattyInfty3 AITalk版を使って、 __合成音声の読み上げを調整する。__  
  Chatty のimlxファイルと、それに付随する画像等（imlxファイルを置いたフォルダの中に自動的に作られる）を、最後の作業者に渡す。
2. 最後の作業者は、DAISY規格やEPUB3規格、EPUB Media Overlays規格、WCAG、WAI-ARIA 等をある程度理解している必要がある。  
  受け取ったChattyのデータから、EPUB3（音声あり）、mp3, 64kbpsとしてエクスポートする。そのEPUB3ファイルはWCAGの基準を満たしていない。  
  自作の自動修正スクリプト等を使ってデータを改変することによって、アクセシブルなデジタル図書としての形式を整える。  
  pagina EPUB-Checker と Ace by DAISY を使って、図書館に納める製品として間違いのないマルチメディアデイジー EPUB3（音声あり）になっていることを確認する。  
  さらに、DAISY2.02マルチメディアデイジーとしてWAVでエクスポートし、それをPRSProでデイジーインポートしてから、mp3, 64kbpsとして音声エクスポートして音声デイジーを完成させる。この回りくどい措置は、ChattyInfty3でエクスポートした音声デイジーをChattyBooksで再生する際、条件により不具合が生じることがあるため（2021年2月25日現在、バグとして報告済）。ただしPRSProでのデイジーインポートでも一部のセクションがごっそり欠ける不具合が生じたことがあるので、最終的にデータが揃っているかを必ずチェックする。

## 2. 音声チームの仕事

AITalkの音声でマルチメディアデイジーを作成する場合は、このステップを飛ばして「3. テキストチームの仕事」に進む。

マルチメディアデイジー図書を国立国会図書館に納品するときは、音声デイジー図書も合わせて納品したいので、 PRS Pro や DR-1 などを使って、従来の自主制作と同じように音声デイジー図書として完成させ、PCMデータのままで Tobi 作業者に渡す。音声チームは mp3 エクスポートをしない。

枠アナウンスのセクションタイトルをマルチメディアデイジーと共用にするため、音声チームは以下のような形になるように製作する。（丸カッコ内）は、原本に応じて異なる。

[録音図書の全国基準](https://www.jla.or.jp/portals/0/html/lsh/zenkokukijyun.html)に反して、セクションタイトルに記入した文字列をすべて読み上げる。例えば、「制作館および製作年月」「著作権処理に関する事項」「デイジー図書奥付」などのセクションタイトルも読み上げる。

これは、 [WCAG](https://ja.wikipedia.org/wiki/Web_Content_Accessibility_Guidelines) 2.1 （ウェブコンテンツ・アクセシビリティ・ガイドライン 2.1）における AAA （トリプルエイ）の基準のうち、 [2.4.10 Section Headings (Level AAA)](https://www.w3.org/WAI/WCAG21/Understanding/section-headings.html) と [3.1.5 Reading Level (Lebel AAA)](https://www.w3.org/WAI/WCAG21/Understanding/reading-level.html) が両立するようなマルチメディアデイジー図書を目指すため。

WCAGの簡単なチェック項目については[クイックリファレンス（英語）](https://www.w3.org/WAI/WCAG21/quickref/)を参照。 WCAG の作成には DAISY Consortium も関わっており、先進諸国ではウェブコンテンツに対して、最低でも WCAG 2.0 の AA （ダブルエイ）の基準を満たすよう求めているところが多い。基準を満たさない場合には、違法であると見なされたり、差別に対する補償金の支払いを命じられたりしている。

国立国会図書館が作成しているテキストデイジー図書は AA を満たすように作られている。 テキストデイジーは音声を含まないので、WCAG 2.1 の [3.1.5 Reading Level (Lebel AAA)](https://www.w3.org/WAI/WCAG21/Understanding/reading-level.html) を満たせないと思われる。 マルチメディアデイジーでは [G79: Providing a spoken version of the text](https://www.w3.org/WAI/WCAG21/Techniques/general/G79.html) によってこの条件を満たすことができるので、他の条件も考慮すれば AAA の準拠も実現可能である。

## 3. テキストチームの仕事

### 3.1. Chatty用テキストの準備

原本が墨字だけしか無い場合は、まず正確なテキストデータを作成する。
デジタル書籍がある場合は、それを購入してDRM解除するほうが格段に早い。

#### 3.1.1. 原本が墨字だけしか無い場合

1. 原本をスキャンする。スキャナは図書館のデジタル工房に2種類あり、好きな方を使う。
2. スキャンして得られるpdfを製品版のOCRにかける。  
  スキャナの付録のOCRは機能が限られているので、製品版OCRを使うほうが正確な結果が得られることが多い。やまびこでは製品版OCR「読取革命 Ver.15」と「e.Typist v.15.0」を所有している。
3. OCRの結果得られるテキストを、原本どおりになるように修正する。  
  修正するためのエディタは何を使っても良いが、多機能のアプリを使う場合は、インデントとスペース・タブの違い、フォントの違い、改行と折返しの違いなど、見た目に騙されないように注意する。  
  保存形式を選べる場合、UTF-8エンコーディング、BOM無し、拡張子txtを選ぶ。  
  余計な改行やスペース、OCRの誤認識に気付くためには、以下の方法が有効。
  - エディタ上で表示するフォントを区別しやすいものに変更する。 [Migu 1M](https://mix-mplus-ipa.osdn.jp/migu/) 推奨。
  - スクリーンリーダーを利用する。
    - Windows10 の場合:  
      Windowsキー（横倒しの台形に細い十字の入ったマーク）とCtrlとEnterキーを同時に押すとナレーターが起動。  
      読み上げ開始したい位置にカーソル（縦棒の点滅）置き、無変換キーとRキーを同時に押すと、その位置から連続読み上げ。  
      Ctrlキーを単独で押すと読み上げ停止。  
      ナレーター機能をやめるには、もう一度WindowsキーとCtrlキーとEnterキーを同時に押す。
    - Chattyのある人は、もちろんナレーターの代わりにChattyを使っても読み上げを聞くことが可能。
    - iPad の場合: [iPadで画面や選択テキストを読み上げる/入力フィードバックを返す - Apple サポート](https://support.apple.com/ja-jp/guide/ipad/ipad9a247097/ipados)
    - macOS の場合: [Macで画面上のテキストを読み上げる - Apple サポート](https://support.apple.com/ja-jp/guide/mac-help/mh27448/11.0/mac/11.0)  
      読み上げ開始したい位置にカーソルを起き、Option キーを押しながら Esc キーを押す。読み上げ開始とともに、コントローラが現れる。
4. 原本にルビがある場合は以下の形でルビを指定する。  
  ルビを付ける文字列の直前に全角縦線`｜`を入れ、その文字列の直後に、ルビの内容を全角二重山括弧`《》`でくくって入れる。この形式のテキストファイルをChattyでインポートすると、ルビの形になる。  
  - 例：  
    `一度｜浅間《あさま》の爆発を`
5. 修正済・ルビ指定済のテキストファイルを1つのフォルダにまとめて、全部一度に ChattyInfty3 にインポートする。画像などは、スキャンしたpdfから抽出して、 ChattyInfty3 編集画面で挿入する。


#### 3.1.2. 原本の電子書籍がある場合

1. 原本と同じ版、またはそれに近い版の Amazon Kindle 版を購入する。
2. 購入した Kindle 書籍の DRM をパソコンで解除する。
3. DRM 解除した azw3 ファイルをパソコンで epub3 に変換する。
4. epub3 を ChattyInfty3 にインポートする。

上の手順4がエラーになる場合は、手順3で epub3 の代わりにテキストファイルに変換したものを ChattyInfty3 にインポートしても良い。その場合、画像などは書籍データから抽出して ChattyInfty3 編集画面で挿入する。


### 3.2. ChattyInfty3 で編集

操作方法については、[「ChattyInfty3 AITalk版の操作方法」](aitalk.html)を参照。

1. テキストに原本と違う部分があれば、修正する。
2. 枠アナウンスとセクション構造は「2. 音声チームの仕事」の節で示した表に従って編集する。「3.3. 音訳者の説明等の形式」で説明する formXX.imlx を使うと、枠アナウンスを作成しやすい。  
  人の声で作成する場合は、音訳者と相談して決めたとおりにセクション構造を作り、枠アナウンスを書き込む。
3. 音訳者として必要な説明を追加する。  
  人の声で作成する場合は、音声デイジーで音訳者が加えた説明に対応するテキストを書き込む。  
  音訳者の説明等は決まった形式で書き込む。詳細は、以下の 「3.3. 音訳者の説明等の形式」を参照。
4. AITalkの音声でデイジー製作する場合は、読み上げを丁寧に調整して、聞いて伝わる読み上げを目指す。  
  人の声でデイジー製作する場合は、合成音声の読み上げが最終的に削除されるので、読み上げに対する編集は不要。
5. マルチメディアデイジーとしてハイライトを区切りたいところに、ハイライト分割を挿入する。  
6. ChattyInfty3 のメニューから、「ファイル＞エクスポート＞全ルビ変換（単語単位）と分かち書き」でエクスポートする。  
  この措置は、 WCAG 2.1 の [3.1.6 Pronunciation (Level AAA)](https://www.w3.org/WAI/WCAG21/Understanding/pronunciation.html) を [H62: Using the ruby element](https://www.w3.org/WAI/WCAG21/Techniques/html/H62.html) によって満たすため。  
  全ルビ変換のメニューが見つからない場合は、[ChattyInftyの概要](https://www.sciaccess.net/jp/ChattyInfty/index.html)ページから、全ルビ生成プラグインをダウンロードしてインストールしてから Chatty を使う。
7. これ以降は分かち書きのファイルの方を編集する。自動生成されたルビが間違っていれば修正する。
8. 以上の編集で間違いがあっても納品前に修正はできるが、その場合は多少の手間がかかる。できるだけここまでの段階で間違いのないように編集する。
9. すべての作業ファイルと元データを次の作業者に渡す。


### 3.3. 音訳者の説明等の形式

暫定的な形式ではあるが、[共有フォルダにある form\*.imlx ファイル](https://drive.google.com/drive/folders/1lQOdyBPgECghazdSIF-zWrg5ktukqLJa?usp=sharing)に示した形式を使う。ファイル名の数字が大きいほうが新しい。

この形式は、「5. Ace by DAISY の検査に合格するための措置」において、 EPUB 3.2 及び WCAG 2.2 の観点から正しいEPUB図書になるように、加工しやすくするための形式である。 スクリプトの改良にしたがって、 formXX.imlx の内容を変更する可能性がある。

#### 使い方
{:.no_toc}

1. form\*.imlx の最新版をダウンロードしておく。
2. 図書のデイジー編集をする imlx ファイルをChattyInfty3で開いて、メニューの「ファイル＞追加読み込み」を使って form\*.imlx のセクションを末尾に入れる。
3. 追加したセクションから、必要な形式をコピー・貼り付けする方法で利用する。
  - 著作権処理のセクションなどはそのまま使えるので、そのセクションを適切な位置に移動すれば良い。
  - いくつかの決まり文句の単語辞書を form\*.imlx 内に登録してあるので、メニューの「設定＞単語辞書をインポート」を使って formXX.imlx から単語をインポートすると、枠アナウンスの読み上げ調整作業を減らすことができる。
4. 必要な形式をコピーし終わったら、 form\*.imlx から追加読み込みした不要なセクションを削除する。


### 3.4. 書誌情報の形式

Chatty の EPUB エクスポートのウィンドウに書誌情報の記入欄はあるが、著者が複数いる場合などにEPUBとして正しい形式で出力できない。また、「5. Ace by DAISY の検査に合格するための措置」の段階で、データ内の書誌情報を含む部分に大幅に手を加えることになるので、そのときについでに自動的に書誌情報を入れることができる。

そのため、書誌情報は[共有フォルダにある「書誌情報書式」](https://drive.google.com/drive/folders/1lQOdyBPgECghazdSIF-zWrg5ktukqLJa?usp=sharing)に示した形式を使う。

#### 使い方
{:.no_toc}

1. 「書誌情報書式」を xlsx などの形式でダウンロードする。ダウンロードしたファイルを、自分で区別しやすいファイル名に変更する。
2. そのファイルを、 Microsoft Excel, Google Sheets, LibreOffice Calc などの好きな表計算アプリで開く。
3. 第1面を正しい情報に書き換える。著者が複数いる場合など、足りない行は挿入して、同じ形式で記入する。該当なしの欄は行ごと削除する。
4. 第1面を tsv ファイルとして保存し、その tsv ファイルを他のデータと一緒に次の作業者に渡す。

## 4. 人間の声の音声デイジーと合成音声のマルチメディアデイジーとの合成

AITalk の音声でマルチメディアデイジーを作成する場合は、このステップを飛ばして、「5. Ace by DAISY の検査に合格するための措置」に進む。

1. Chatty から EPUB3（音声あり）でエクスポート。音声データは Tobi で置き換えてしまうので、圧縮方式はどれでも良い。音声変換設定をWAVにしておくと、データ圧縮が無い分だけ処理が速い。  
  どうせ音声を入れ替えるのに「音声あり」でエクスポートする理由は、音声なしだとハイライト区切りの情報が消されてしまうため。
2. Tobi を起動し、File > Import から、 Chatty で出力した EPUB3 を開く。  
  「EPUB-Check?」と聞かれるが、No を選ぶ。  
  Project audio format として Hz22050 を選ぶ。 これは録音図書の全国基準でそう決められているので選択の余地がない。
3. Project Spine の画面が出る。セクションが縦に並んでいる。1つのセクションを選んでOKする。普通は上から順に選ぶと作業しやすい。  
4. 右の枠、上部アイコンのすぐ下にある「body」をクリックすると、このセクションの音声全体が下の欄に見える。 Ctrl A で全部の音声を選択。 Del キーで音声を削除。
5. 左下の青くて四角いフォルダのアイコンをクリック。音声チームから受け取ったフォルダを開き、音声ファイル a000001.wav などから適切な音声を選んで開く。
  - 音声チームが最後にビルドブックしてあれば、すべてのレベルの1つのセクションにつき、1つの音声ファイルができている。それらの音声ファイルは、セクションの順番どおりの番号でファイル名が付いている。
  - Tobi で開いているセクションは、レベル1のセクションごとに分割されたものである。2レベル以上のセクション階層を付けた図書の場合は、Tobi で開いているセクションに対して、音声チームからの複数の音声ファイルを当てることになる。
6. 取り込んだ音声を聞きながら、フレーズの変わり目で`Ctrl`を押しながら改行キーを押すと、テキストのハイライトと音声が同期していく。タイミングがずれた場合は、後でマウスを使って直す。
7. 今開いているセクションの音声をすべて当て終わったら、 Ctrl S で保存してから、 Ctrl U で Project Spine を開き、次のセクションを選んで同様の作業をする。
8. すべてのセクションの音声を当てて保存したら Tobi を閉じ、作業ファイルと元データを次の作業者に渡す。

## 5. EPUB3規格とアクセシビリティの検査に合格するための措置

 EPUB 3.2 及び WCAG 2.2 の観点から正しいEPUB図書になるように加工するスクリプト等を自分で書き、それを使って、 Tobi から出力したファイルを変換する。
変換後のファイルを [pagina EPUB-Checker](https://www.pagina.gmbh/produkte/epub-checker/) と [Ace by DAISY](https://daisy.github.io/ace/) に通して検査し、両方に合格したらUSBメモリに入れ、国立国会図書館に郵送納品する。
